# PRODUCT.md - 侵权检测训练图库生成器

## 产品愿景

构建一个可验证、可迭代的训练图库生成系统，为侵权检测 AI 模型准备带有来源、标签、相似度分层和生成参数的图片数据集。系统只负责图库生成、导入、增强、标注和质检，不负责模型训练本身。系统先聚焦外观设计专利，再扩展到其他侵权类型。

## 目标用户

- 侵权检测 AI 模型训练者
- 知识产权保护平台开发者
- 电商平台内容审核系统

## 侵权类型覆盖

| 编号 | 侵权类型 | 说明 |
|------|----------|------|
| T1 | 盗图 | 直接复制或轻微修改他人图片 |
| T2 | 外观设计专利 | 产品外观与已注册专利设计实质性相似 |
| T3 | 美术作品著作权 | 绘画、插画、设计稿被复制或改编 |
| T4 | 文字商标权 | 未经授权使用已注册商标文字 |
| T5 | 图案商标权 | 未经授权使用已注册商标图形 |
| T6 | 肖像权 | 未经授权使用他人肖像 |

## 当前产品原则

- 训练数据必须可追溯：每张样本都记录来源、对应注册图、生成参数和相似度分层。
- placeholder 只能用于测试，不能作为生产训练数据。
- 样本集应覆盖多个相似度区间，尤其保留边界样本，避免模型只学习极端正负例。
- 在线公开数据采集存在稳定性和合规风险，应独立验证后再进入主流程。
- 本项目不实现模型训练、训练框架封装、训练/验证/测试切分或训练调度。

## 数据结构

```
datasets/
├── registry/
│   └── {侵权类型}/
│       └── patent_{注册ID}.png
├── training/
│   └── {侵权类型}/
│       ├── positive/
│       │   └── positive_{注册ID}_{序号}.png
│       └── negative/
│           └── negative_{样本ID}_{序号}.png
└── metadata.csv
```

## 成功标准

### Phase 0.1 - 可信本地 MVP

- 仅实现外观设计专利（T2）。
- 支持从本地 manifest 或 ID 文件导入注册图，归一化为 PNG、512x512、RGB。
- 生产模式不允许静默生成 placeholder；测试模式可以显式启用 placeholder。
- 精确生成注册图 >=5 张、训练样本正样本 20 张、负样本 20 张。
- metadata 记录 label、registry_id、source、transformations JSON、similarity_score、similarity_band。
- 正样本覆盖 `mid` 和 `positive` 两个相似度分层，负样本进入 `negative` 分层。

### Phase 0.2 - 在线数据源 Spike

- 验证 CNIPA、WIPO 或其他公开数据源的访问方式、图片提取方式、反爬限制和合规约束。
- 只有在真实下载与元数据可靠后，才把在线采集接入主流程。

### Phase 1.0 - 图库交付增强

- 扩展更强的相似度估计方法。
- 支持多种图库导出格式和批次质检报告。
- 支持人工复核标记和样本重标。

### Phase 2.0 - 规模化

- 扩展全部 6 种侵权类型。
- 每类 1000 张以上训练样本。
- 接入 ComfyUI 做 img2img 增强真实感。
